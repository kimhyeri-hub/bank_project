# 📋 Guardian AI: 기능 요구사항 명세서
### Requirements Specification — MoSCoW 기법 적용

---

## 1. 개요

본 문서는 Guardian AI MVP 개발을 위한 기능 요구사항을 **MoSCoW 우선순위 기법**으로 분류한다.

- **M — Must Have**: 반드시 구현해야 하는 핵심 기능 (없으면 제품 성립 불가)
- **S — Should Have**: 중요하지만 초기 릴리스에 없어도 운영 가능
- **C — Could Have**: 있으면 좋지만 시간/자원 여유 시 구현
- **W — Won't Have**: 이번 범위(MVP)에서는 제외

---

## 2. Must Have — 핵심 기능 (6주 내 필수 완성)

### M-01. PII(개인식별정보) 자동 마스킹

| 항목 | 내용 |
|------|------|
| 기능 ID | F-PII-001 |
| 우선순위 | Must Have |
| 설명 | 사용자가 입력하거나 붙여넣은 텍스트에서 PII를 자동 탐지·마스킹 |
| 입력 | 자유 텍스트 (직접 입력, 클립보드 붙여넣기) |
| 처리 | 정규식 + AI 기반 PII 엔티티 인식 |
| 출력 | 마스킹 처리된 텍스트 (`홍길동` → `***`, `010-xxxx-xxxx` → `010-****-****`) |
| 탐지 대상 | 이름, 전화번호, 이메일, 주민등록번호, 신용카드 번호, 주소, 계좌번호 |
| 비기능 요건 | 처리 시간 3초 이내, 민감 데이터 서버 미전송 원칙 |

**세부 요구사항:**
- [ ] 한국어 이름 패턴 인식 (2~4자 한글)
- [ ] 국내 전화번호 형식 (010/011/016/017/018/019)
- [ ] 이메일 주소 (RFC 표준)
- [ ] 주민등록번호 13자리 패턴
- [ ] 신용카드 번호 (Luhn 알고리즘 검증)
- [ ] 마스킹 강도 조절 옵션 (부분 마스킹 / 완전 마스킹)
- [ ] 마스킹 전/후 비교 화면 제공

### M-02. 약관 분석 및 위험 조항 요약

| 항목 | 내용 |
|------|------|
| 기능 ID | F-TOS-001 |
| 우선순위 | Must Have |
| 설명 | 서비스 이용약관 텍스트를 AI로 분석하여 위험 조항 요약 및 위험도 평가 제공 |
| 입력 | 텍스트 직접 입력, 텍스트 파일(.txt) 업로드 |
| 처리 | Claude API를 이용한 NLP 분석 |
| 출력 | 위험 조항 목록, 위험도 점수(1~10), 핵심 요약 (3~5줄), 개선 권고 |

**세부 요구사항:**
- [ ] 위험 카테고리 분류: 데이터 수집·판매, 제3자 공유, 일방적 계약 변경, 책임 면제, 자동 결제
- [ ] 위험도 3단계 시각화: 🟢 안전 / 🟡 주의 / 🔴 위험
- [ ] 조항별 원문 + AI 해석 나란히 표시
- [ ] 분석 결과 텍스트 공유 기능
- [ ] 처리 로딩 상태 표시 (분석 중 UX)

---

### M-03. 피싱 URL/링크 탐지

| 항목 | 내용 |
|------|------|
| 기능 ID | F-PHISH-001 |
| 우선순위 | Must Have |
| 설명 | URL 또는 SMS/이메일 원문을 입력받아 피싱 여부 판별 |
| 입력 | URL 직접 입력, 전체 메시지 텍스트 붙여넣기 |
| 처리 | 패턴 분석 + 외부 위협 DB 조회 + AI 문맥 분석 |
| 출력 | 피싱 여부 (True/False), 위험도 점수, 위험 요소 상세 설명 |

**세부 요구사항:**
- [ ] URL 구조 분석 (도메인 유사성, 단축 URL 추적, HTTPS 여부)
- [ ] 한국어 스미싱 문자 패턴 인식
- [ ] 긴급성 유도 문구 탐지 ("지금 바로", "48시간 내", "계정 정지")
- [ ] 사칭 기관명 탐지 (국세청, 카카오, 네이버, 금융감독원 등)
- [ ] 위험 URL 결과 화면에 명확한 경고 UI 표시
- [ ] 분석 결과 캡처/공유 기능 (주변에 알리기)

- Flutter `intl` 패키지 활용

- 타 앱에서 텍스트/URL 공유 시 Guardian AI로 바로 분석
- Android Intent / iOS Share Extension

### C-05. 위협 DB 커뮤니티

- 사용자 신고 기반 피싱 URL DB 공유
- 익명 기여 방식

## 5. Won't Have — MVP 범위 제외

| 기능 | 제외 이유 |
|------|---------|
| 실시간 화면 모니터링 | 플랫폼 권한·배터리 이슈, MVP 범위 초과 |
| 사용자 계정/로그인 시스템 | 백엔드 인프라 필요, 6주 내 불가 |
| VPN 연동 | 기술 복잡도 과도 |
| 기업용 B2B 대시보드 | 타겟 범위 초과 |
| 자동 약관 웹 크롤링 | 법적·기술적 리스크 |
| 결제·구독 기능 | MVP 이후 단계 |

## 6. 비기능 요구사항 (Non-Functional Requirements)

### 성능 (Performance)
- PII 마스킹: 로컬 처리 기준 **3초 이내**
- 약관 분석: API 응답 포함 **30초 이내**
- 피싱 탐지: **10초 이내**
- 앱 콜드 스타트: **3초 이내**

### 보안 (Security)
- 분석 대상 원문 텍스트는 서버에 저장 불가 (사용 후 즉시 폐기)
- API 키는 환경변수 또는 Flutter Secure Storage 관리
- 로컬 저장 데이터는 암호화 옵션 제공

### 사용성 (Usability)
- 핵심 기능까지 탭 **3번 이내** 도달
- 텍스트 입력 → 결과 확인까지 **2단계 이내**
- 에러 메시지는 사용자 친화적 언어 사용 (기술 용어 배제)

### 호환성 (Compatibility)
- Android 8.0 (API 26) 이상
- iOS 13.0 이상
- Flutter 3.x 최신 안정 버전
