## 2. 기술적 도전 과제 (Technical Challenges)

### Challenge 1. 🧠 멀티모달 AI 파이프라인 설계
**도전 내용:**  
약관 분석, PII 탐지, 피싱 판별이라는 세 가지 전혀 다른 NLP 태스크를 하나의 Claude API 엔드포인트 위에서 운영하되, 각 태스크에 최적화된 **독립 프롬프트 엔지니어링 체계**를 설계한다.

**기술적 난이도:**
- 단순 API 호출이 아닌, 구조화된 JSON 응답 강제 (Function Calling 패턴)
- Few-shot 프롬프팅으로 일관된 결과 품질 확보
- 긴 약관 텍스트 처리를 위한 **청킹(Chunking) + 요약 병합 전략**
- 프롬프트 버전 관리 시스템 구축

**성과 지표:**
- [ ] 약관 분석 정확도: 전문가 검토 대비 주요 위험 조항 탐지율 **80% 이상**
- [ ] 피싱 판별 정확도: 테스트 케이스 대비 **True Positive Rate 85% 이상**
- [ ] PII 탐지 정확도: 테스트 데이터셋 기준 **Precision 90% 이상**

---

### Challenge 2. 🔐 Privacy-by-Design 아키텍처
**도전 내용:**  
개인정보 보호 앱이 역설적으로 개인정보를 수집하는 문제를 원칙적으로 차단하는 **Privacy-by-Design** 아키텍처를 구현한다.

**기술적 구현:**
- 민감 텍스트 원문의 **서버 비전송 원칙** (On-Device 전처리 후 익명화)
- PII 마스킹은 100% 로컬 처리 (API 호출 불필요)
- API 전송 전 자동 PII 익명화 레이어 삽입
- 로컬 히스토리 데이터 AES 암호화 저장

**성과 지표:**
- [ ] 네트워크 패킷 캡처 시 원문 PII 데이터 **0건** 전송 확인
- [ ] 로컬 DB 파일 직접 열기 시 **암호화 적용** 확인
- [ ] 앱 삭제 시 모든 데이터 완전 삭제 검증

### Challenge 3. 🌐 한국어 특화 NLP 처리
**도전 내용:**  
기존 영어 중심 피싱 탐지 및 PII 인식 모델의 한계를 극복하여, **한국어 특화 패턴 인식 엔진**을 자체 구축한다.

**기술적 구현:**
- 한국어 이름 패턴: 성씨 DB + 음절 패턴 기반 인식 (단순 정규식 초월)
- 국내 스미싱 공식 패턴 DB 자체 구축 (금융감독원 사례 기반)
- 사칭 기관명 한국어 유사도 매칭 (레벤슈타인 거리 활용)
- 한국어 약관의 법률 용어 → 일상 언어 변환 프롬프트

**성과 지표:**
- [ ] 한국어 이름 탐지: 일반 정규식 대비 **오탐율 50% 감소**
- [ ] 스미싱 탐지: 국내 실제 스미싱 사례 50건 테스트 시 **정탐율 80% 이상**
- [ ] 약관 번역 가독성: 사용자 평가 5점 만점 기준 **4.0점 이상**

### Challenge 4. ⚡ 실시간 반응형 UX + 성능 최적화
**도전 내용:**  
AI API 응답 대기 시간(Latency) 문제를 해결하여, 사용자가 **지루함을 느끼지 않는** 실시간 반응형 UX를 구현한다.

**기술적 구현:**
- **스트리밍 응답(SSE)** 적용: 분석 결과를 토큰 단위로 실시간 표시
- 로컬 처리(정규식) 결과를 먼저 보여주고, AI 결과로 점진적 업데이트 (Progressive Enhancement)
- LRU 캐시 적용: 동일 URL/약관 재분석 시 API 호출 생략
- Flutter Isolate 활용: 무거운 텍스트 처리를 별도 스레드에서 실행

**성과 지표:**
- [ ] 앱 콜드 스타트: **3초 이내**
- [ ] PII 마스킹 (로컬): **1초 이내** (10,000자 기준)
- [ ] 약관 분석 첫 결과 표시: **5초 이내** (스트리밍 기준)
- [ ] 메모리 사용량: **150MB 이하** (일반 사용 시)

### Challenge 5. 🧪 테스트 자동화 및 품질 보증
**도전 내용:**  
졸업 작품 수준을 넘어, 실제 프로덕션 품질의 **자동화 테스트 스위트**를 구축한다.

**기술적 구현:**
- 단위 테스트: PII 탐지 엔진 (100+ 테스트 케이스)
- 통합 테스트: API 연동 모킹(Mocking)을 통한 오프라인 테스트
- 위젯 테스트: 주요 화면 골든 파일 테스트
- 코드 커버리지 측정 및 보고서 생성

**성과 지표:**
- [ ] 단위 테스트 커버리지: **70% 이상**
- [ ] CI 파이프라인 자동 실행: PR마다 테스트 통과 필수
- [ ] 테스트 케이스 수: **50개 이상**
